# 🎟️ Ticket Manager

Ελαφριά εφαρμογή έκδοσης εισιτηρίων & box‑office για **θέατρα, κινηματογράφους, μουσεία και εκδηλώσεις** — με τοπική λειτουργία (ταμείο) και **online κρατήσεις** στο cloud. Αντικαθιστά παλαιότερη εφαρμογή PowerBuilder/Sybase.

---

## Επισκόπηση

Δύο κόσμοι που συνεργάζονται:

- **Τοπικό σύστημα (ταμείο):** Node.js server + SQLite, με web περιβάλλον στο τοπικό δίκτυο. Έκδοση εισιτηρίων, επιλογή θέσεων, ταμείο, αναφορές, έλεγχος εισόδου, θερμική εκτύπωση, φορολογικά.
- **Online κρατήσεις (cloud):** δημόσια σελίδα κράτησης, πληρωμή με κάρτα (Viva), ηλεκτρονικό εισιτήριο με QR στο email. Συγχρονισμός με το τοπικό σύστημα.

---

## Βασικά χαρακτηριστικά

### Έκδοση εισιτηρίων
- **POS (σειριακή έκδοση):** κουμπιά τύπων εισιτηρίων, ποσότητα, άμεση έκδοση με Μετρητά/Κάρτα ή καλάθι.
- **Αίθουσες / Θέσεις:** επιλογή ημερομηνίας → θέαμα → οπτική επιλογή θέσεων πάνω στον χάρτη της αίθουσας → έκδοση.
- Ρυθμιζόμενη εμφάνιση οθονών (μόνο POS, μόνο Αίθουσες, ή και τα δύο).

### Αίθουσες & σχεδιασμός
- Σχεδιαστής κάτοψης: σειρές/στήλες, **διάδρομοι & κενά**, αυτόματη αρίθμηση θέσεων (π.χ. Α1–Κ20).
- Κλείδωμα δομής μόλις εκδοθούν εισιτήρια (ασφάλεια ακεραιότητας) — ενεργοποίηση/απενεργοποίηση αντί διαγραφής.

### Πρόγραμμα θεαμάτων
- Πολλαπλά **ωριαία διαστήματα × εύρη ημερομηνιών** (κάθε συνδυασμός = ξεχωριστό θέαμα).
- Επαναλαμβανόμενα θεάματα σε εύρος ημερομηνιών, με διαθεσιμότητα θέσεων **ανά ημερομηνία**.
- Αναζήτηση, ταξινόμηση, ενεργοποίηση/απενεργοποίηση.

### Τύποι εισιτηρίων & αρίθμηση
- Τύποι με τίτλο, τιμή, ΦΠΑ, χρώμα, προεπιλεγμένο τρόπο πληρωμής, σειρά εμφάνισης.
- **Αρίθμηση**: ενιαία (ένας μετρητής) ή ανά τύπο (πρόθεμα + δικός του μετρητής).

### Πληρωμές
- Μετρητά / Κάρτα.
- **Viva Payments** ως POS (κάρτα): Smart Checkout και Cloud Terminal — η πληρωμή «σηκώνεται» μέσα από τη σελίδα έκδοσης.

### Έλεγχος εισόδου (check‑in)
- Σάρωση **QR** ή πληκτρολόγηση αριθμού· σήμανση «μπήκε», ζωντανά στατιστικά & λίστα.
- **Χρονικό παράθυρο**: δέχεται μόνο εισιτήρια του θεάματος που «τρέχει» (ανοίγει Χ λεπτά πριν την έναρξη, κλείνει στη λήξη) — ρυθμιζόμενο, ανεξαρτήτως αίθουσας.
- Λειτουργεί σε όλο το τοπικό δίκτυο (scanner ως πληκτρολόγιο).

### Ταμείο & Αναφορές
- Ταμείο ημέρας (μετρητά/κάρτα), λίστα εκδομένων εισιτηρίων, επανεκτύπωση.
- Αναφορές: τζίρος, ΦΠΑ, εισιτήρια, ανά θέαμα/αίθουσα/τύπο, ημερήσιο γράφημα, εξαγωγή CSV.
- **Ανάλυση καναλιού: Online vs Τοπικά** (τζίρος, πωλήσεις, εισιτήρια).

### Πελατολόγιο
- Στοιχεία πελατών (επωνυμία, ΑΦΜ, διεύθυνση, email, τηλέφωνα) για marketing & φορολογικά.
- Προεπιλεγμένος «Πελάτης Λιανικής».

### Εκτύπωση
- **Θερμικοί ESC/POS 58/80mm** και **Zebra ZPL**.
- Ελληνικά (κωδικοσελίδες CP737/Windows‑1253), ρυθμιζόμενα μεγέθη, **QR** στο εισιτήριο.
- Παραμετρικές φόρμες (Κεφαλίδα–Λεπτομέρειες–Υποσέλιδο) με placeholders & inline tags.
- Σταθμοί ταμείου → αντιστοίχιση εκτυπωτή.

### Φορολογικά
- Τρόποι έκδοσης: ανενεργό / μόνο εισιτήριο / ταμειακή (αρχείο) / **πάροχος ηλεκτρονικής έκδοσης**.
- Ενσωμάτωση **RapidSign / MyMat** (ΑΠΥ) — με ρυθμίσεις & δοκιμή σύνδεσης.
- Ρυθμιζόμενη νομική σημείωση (π.χ. «Δεν αποτελεί φορολογικό παραστατικό»).

### Online κρατήσεις (cloud)
- Δημόσια σελίδα κράτησης με **ημερολόγιο**: επιλογή ημέρας → θεάματα της ημέρας → επιλογή θέσεων (ίδιος χάρτης με του ταμείου, με διαδρόμους) → πληρωμή.
- **Πληρωμή μόνο με κάρτα (Viva)**. Μετά την πληρωμή, **ηλεκτρονικό εισιτήριο με QR** στο email (και ως σύνδεσμος στο δικό σας domain) + PDF.
- Κλείσιμο online πωλήσεων ανά ημέρα/ώρα.

### Συγχρονισμός τοπικού ↔ cloud
- Δημοσίευση θεάματος για **εύρος ημερομηνιών** — μεταφέρει θέσεις, τιμές, σχέδιο.
- **Αμφίδρομος**: ανεβάζει πωλήσεις ταμείου (πιασμένες θέσεις) και κατεβάζει online πωλήσεις.
- Οι **online πωλήσεις γίνονται τοπικές πωλήσεις + εισιτήρια** με πωλητή «web» → εμφανίζονται σε ταμείο, αναφορές, λίστες και λειτουργεί ο έλεγχος εισόδου.
- **Απόσυρση**: κατεβάζει τις πωλήσεις τοπικά και διαγράφει το θέαμα από το cloud (μικρή online βάση).

### Ρόλοι χρηστών
- **Διαχειριστής** (πλήρης πρόσβαση), **Ταμίας** (έκδοση/ταμείο/είσοδος), **Ελεγκτής** (μόνο είσοδος), **web** (εικονικός πωλητής για online).

### Ασφάλεια δεδομένων
- Έλεγχος ακεραιότητας (δεν διαγράφονται αίθουσες/θεάματα/πελάτες με ιστορικό).
- **Backup** βάσης με ένα κλικ + λήψη.

## Αρχιτεκτονική / Τεχνολογίες

| Στρώμα | Τεχνολογία |
|---|---|
| Τοπικός server | Node.js (≥22), Fastify, **node:sqlite** (χωρίς native build), JWT |
| Τοπικό UI | React + Vite + Tailwind |
| Εκτύπωση | ESC/POS (iconv‑lite), Zebra ZPL, QR |
| Cloud βάση | Supabase (Postgres + RLS) |
| Cloud λογική | Supabase Edge Functions (Deno) |
| Booking site | React + Vite (φιλοξενία σε Cloudflare) |
| Πληρωμές | Viva Payments (Smart Checkout / Cloud Terminal) |
| Email | Resend (transactional) |
| Φορολογικά | RapidSign / MyMat |

## Εγκατάσταση & Εκτέλεση

> Απαιτείται **Node.js ≥ 22** (ιδανικά 24). Η βάση είναι ενσωματωμένη `node:sqlite` — δεν χρειάζεται εξωτερική εγκατάσταση.

```bash
npm install            # εγκατάσταση εξαρτήσεων (workspaces: server + client)
npm run seed           # αρχικοποίηση βάσης + προεπιλεγμένοι χρήστες
```

### Ανάπτυξη (development)
```bash
npm run dev            # API :3001 + UI (Vite) :5173
# Άνοιγμα: http://localhost:5173
```

### Παραγωγή (production)
```bash
npm run build          # build του client (minified)
npm start              # API + σερβίρισμα UI στο :3001
# Άνοιγμα: http://localhost:3001
```

**Προεπιλεγμένοι χρήστες:** `admin/admin` (διαχειριστής), `user` (ταμίας, χωρίς κωδικό), `checker` (ελεγκτής, χωρίς κωδικό).

## Εγχειρίδιο χρήσης

### 1. Πρώτη ρύθμιση (Διαχειριστής)
1. Σύνδεση ως `admin`.
2. **Ρυθμίσεις → Επιχείρηση:** στοιχεία επιχείρησης (επωνυμία, ΑΦΜ, ΔΟΥ…), τρόπος λειτουργίας έκδοσης.
3. **Ρυθμίσεις → Τύποι Εισιτηρίων:** τύποι, τιμές, και ρυθμίσεις **αρίθμησης** + **παραθύρου check‑in**.
4. **Ρυθμίσεις → Εκτυπωτές:** εκτυπωτές, σταθμοί, τρόπος έκδοσης/φορολογικά, πάροχος (RapidSign).
5. **Αίθουσες:** σχεδίαση κάτοψης & θέσεων.
6. **Πρόγραμμα:** δημιουργία θεαμάτων (ώρες × ημερομηνίες) και τύποι ανά θέαμα.

### 2. Έκδοση εισιτηρίων (Ταμίας)
- **POS:** διάλεξε ποσότητα → πάτησε τύπο εισιτηρίου → έκδοση (Μετρητά/Κάρτα).
- **Αίθουσες:** ημερομηνία → θέαμα → επιλογή θέσεων → ΕΚΔΟΣΗ. Με κάρτα & ενεργό Viva, ανοίγει η πληρωμή POS.

### 3. Έλεγχος εισόδου
- Οθόνη **Είσοδος**: σάρωσε το QR (ή πληκτρολόγησε αριθμό) → πράσινο «ΕΓΚΥΡΟ». Δέχεται μόνο εισιτήρια του τρέχοντος θεάματος.
- **⟳ Συγχρονισμός Cloud**: κατεβάζει τις online πωλήσεις πριν την είσοδο.

### 4. Ταμείο & Αναφορές
- **Ταμείο:** σύνολα ημέρας, λίστα εισιτηρίων, επανεκτύπωση.
- **Αναφορές:** εύρος ημερομηνιών, κάρτες (τζίρος/ΦΠΑ/εισιτήρια/**Online**/Τοπικά), πίνακες ανά θέαμα/αίθουσα/τύπο, CSV.

### 5. Online κρατήσεις
1. **Ρυθμίσεις → Online Ρυθμίσεις:** σύνδεση Cloud (URL + κλειδί) και POS (Viva).
2. **Online:** διάλεξε ημερομηνία → θέαμα → εύρος δημοσίευσης & ώρα κλεισίματος → **Δημοσίευση**.
3. Ο πελάτης κλείνει από τη δημόσια σελίδα, πληρώνει με κάρτα, λαμβάνει e‑εισιτήριο.
4. **Συγχρονισμός**: οι online πωλήσεις περνούν τοπικά (πωλητής «web»).
5. **Απόσυρση**: κατεβάζει πωλήσεις & διαγράφει το θέαμα από το cloud.

## Άδεια / Σημειώσεις

Ιδιωτικό έργο. Τα διαπιστευτήρια (Viva, RapidSign, Supabase) ορίζονται από τις ρυθμίσεις/secrets και δεν αποθηκεύονται στο αποθετήριο.
